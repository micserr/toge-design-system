# แนวทางเนื้อหา

ภาษาที่สม่ำเสมอสร้างความไว้วางใจและลดความสับสน แนวทางเหล่านี้ครอบคลุมวิธีการเขียนข้อความ UI ในผลิตภัณฑ์ Sprout ทุกตัว

## น้ำเสียงและโทนเสียง

**น้ำเสียง** สม่ำเสมอในทุกผลิตภัณฑ์:

| หลักการ | คำอธิบาย |
|---|---|
| **ชัดเจน** | พูดในสิ่งที่คุณหมายความ หลีกเลี่ยงศัพท์เทคนิคและความกำกวม |
| **กระชับ** | ใช้คำให้น้อยที่สุดเท่าที่จำเป็น เคารพเวลาของผู้ใช้ |
| **เป็นประโยชน์** | นำทางผู้ใช้ไปสู่เป้าหมาย เสนอขั้นตอนถัดไป |
| **เป็นมืออาชีพ** | เป็นมิตรแต่ไม่เป็นกันเอง ไม่ใช้คำสแลง |

**โทนเสียง** แตกต่างตามบริบท:

| บริบท | โทนเสียง | ตัวอย่าง |
|---|---|---|
| สำเร็จ | อบอุ่น กระชับ | "บันทึกการเปลี่ยนแปลงแล้ว" |
| ข้อผิดพลาด | ตรงไปตรงมา เป็นประโยชน์ | "ไม่สามารถบันทึกได้ ตรวจสอบการเชื่อมต่อของคุณแล้วลองอีกครั้ง" |
| คำเตือน | ระมัดระวัง ชัดเจน | "การดำเนินการนี้ไม่สามารถย้อนกลับได้" |
| สถานะว่าง | ให้กำลังใจ | "ยังไม่มีพนักงาน เพิ่มพนักงานคนแรกของคุณเพื่อเริ่มต้น" |
| กำลังโหลด | เป็นกลาง | "กำลังโหลดพนักงาน..." |
| การยืนยัน | จริงจัง เฉพาะเจาะจง | "ลบ John Doe? การดำเนินการนี้จะลบระเบียนที่เกี่ยวข้องทั้งหมด" |

## ศัพท์เฉพาะ

คำศัพท์เฉพาะของ Sprout ที่ใช้อย่างสม่ำเสมอ:

| ใช้ | ไม่ใช้ |
|---|---|
| Employee | Staff, Worker, User |
| Department | Division, Team |
| Leave | Vacation, PTO, Absence |
| Overtime | OT (หลีกเลี่ยงตัวย่อใน UI) |
| Payroll | Pay run |
| Cutoff | Pay period, Cycle |
| Approver | Manager (ยกเว้นเป็นตำแหน่งจริง) |
| Effective date | Start date (ยกเว้นเป็นวันเริ่มต้นจริงๆ) |

## ป้ายปุ่ม

| การดำเนินการ | ป้าย | ไม่ใช้ |
|---|---|---|
| สร้างใหม่ | "Add [noun]" | "Create", "New", "+ Add" |
| บันทึก | "Save" หรือ "Save changes" | "Submit", "Done", "OK" |
| ลบ | "Delete [noun]" | "Remove", "OK", "Yes" |
| ยกเลิก | "Cancel" | "Back", "Close", "Never mind" |
| ถัดไป | "Next" หรือ "Continue" | "Go", "Proceed" |
| ย้อนกลับ | "Back" | "Previous", "Return" |
| ส่งออก | "Export" หรือ "Download" | "Get file" |
| กรอง | "Apply filters" | "Filter", "Search", "Go" |
| รีเซ็ต | "Clear filters" | "Reset", "Clear all" |

### กฎของปุ่ม

- ใช้**คำกริยา** -- ปุ่มคือการดำเนินการ ไม่ใช่คำอธิบาย
- **เจาะจง**สำหรับการดำเนินการที่ทำลายข้อมูล -- "Delete employee" ไม่ใช่แค่ "Delete"
- สูงสุด **3 คำ** สำหรับปุ่มมาตรฐาน
- ป้ายปุ่มหลักต้องตรงกับคำกริยาในชื่อกล่องโต้ตอบ

## ข้อความข้อผิดพลาด

Sprout ใช้กรอบการส่งข้อความสองแบบ ขึ้นอยู่กับว่าข้อผิดพลาด**เกิดขึ้นแล้ว**หรือผู้ใช้**กำลังจะทำสิ่งที่ย้อนกลับไม่ได้**

### การกู้คืนจากข้อผิดพลาด (Error Recovery)

เมื่อเกิดข้อผิดพลาดแล้ว ให้ปฏิบัติตามสูตร 4 ขั้นตอน:

| ขั้นตอน | วัตถุประสงค์ | ตัวอย่าง |
|---|---|---|
| 1. บอกว่าเกิดอะไรขึ้น | ระบุปัญหาอย่างชัดเจน | "ไม่สามารถบันทึกการเปลี่ยนแปลงของคุณได้" |
| 2. บอกสาเหตุ | ให้บริบท (ถ้าทราบ) | "เซิร์ฟเวอร์ไม่พร้อมใช้งานชั่วคราว" |
| 3. ช่วยแก้ไข | เสนอขั้นตอนถัดไปที่ดำเนินการได้ | "กรุณาลองอีกครั้งในอีกสักครู่" |
| 4. เสนอทางออก | เสนอทางเลือกหรือการสนับสนุน | "หากปัญหายังคงอยู่ ติดต่อฝ่ายสนับสนุน" |

**ตัวอย่างเต็ม:**

> **ไม่สามารถบันทึกการเปลี่ยนแปลงของคุณได้** เซิร์ฟเวอร์ไม่พร้อมใช้งานชั่วคราว กรุณาลองอีกครั้งในอีกสักครู่ หากปัญหายังคงอยู่ ติดต่อฝ่ายสนับสนุน

**ตัวอย่างเพิ่มเติม:**

| สถานการณ์ | ข้อความ |
|---|---|
| ข้อผิดพลาดเครือข่าย | "ไม่สามารถเชื่อมต่อได้ การเชื่อมต่ออินเทอร์เน็ตของคุณอาจขาดหาย ตรวจสอบการเชื่อมต่อแล้วลองอีกครั้ง" |
| ไม่มีสิทธิ์ | "คุณไม่มีสิทธิ์ดำเนินการนี้ การดำเนินการนี้ต้องมีสิทธิ์ผู้ดูแลระบบ ติดต่อผู้ดูแลระบบเพื่อขอสิทธิ์" |
| ข้อขัดแย้ง | "ระเบียนนี้ถูกอัปเดตโดยผู้อื่นขณะที่คุณกำลังแก้ไข รีเฟรชหน้าเพื่อดูเวอร์ชันล่าสุด การเปลี่ยนแปลงที่ยังไม่ได้บันทึกจะหายไป" |
| ข้อผิดพลาดเซิร์ฟเวอร์ | "เกิดข้อผิดพลาดจากฝั่งเรา กรุณาลองอีกครั้งในภายหลัง หากปัญหายังคงอยู่ ติดต่อฝ่ายสนับสนุน" |

### การป้องกันข้อผิดพลาด (Error Prevention)

ก่อนดำเนินการที่ทำลายข้อมูลหรือย้อนกลับไม่ได้ ให้ปฏิบัติตามสูตร 4 ขั้นตอน:

| ขั้นตอน | วัตถุประสงค์ | ตัวอย่าง |
|---|---|---|
| 1. บอกว่ากำลังจะทำอะไร | ระบุการดำเนินการที่กำลังจะเกิดขึ้น | "คุณกำลังจะลบ API key ที่เข้ารหัส" |
| 2. บอกว่าจะเกิดอะไรขึ้น | อธิบายผลลัพธ์ทันที | "การดำเนินการนี้ไม่สามารถย้อนกลับได้" |
| 3. อธิบายผลกระทบ | อธิบายผลกระทบต่อเนื่อง | "จะส่งผลกระทบต่อบริการทั้งหมดที่ใช้คีย์นี้" |
| 4. เสนอทางออก | ให้ความช่วยเหลือหรือการยกเลิก | "หากไม่แน่ใจหรือต้องการความช่วยเหลือ กรุณาติดต่อฝ่ายสนับสนุน" |

**ตัวอย่างเต็ม:**

> **ยืนยันการลบ API Key**
>
> คุณกำลังจะลบ API key ที่เข้ารหัส การดำเนินการนี้ไม่สามารถย้อนกลับได้และจะส่งผลกระทบต่อบริการทั้งหมดที่ใช้คีย์นี้
>
> หากไม่แน่ใจหรือต้องการความช่วยเหลือ กรุณาติดต่อฝ่ายสนับสนุน
>
> \[ยกเลิก\] \[ลบ API Key\]

**ตัวอย่างเพิ่มเติม:**

| สถานการณ์ | ชื่อ | เนื้อหา |
|---|---|---|
| ลบพนักงาน | "ลบ John Doe?" | "การดำเนินการนี้จะลบ John Doe และระเบียนที่เกี่ยวข้องทั้งหมด (ลา, เงินเดือน, ผลการปฏิบัติงาน) อย่างถาวร ไม่สามารถย้อนกลับได้" |
| การดำเนินการจำนวนมาก | "เก็บถาวรพนักงาน 15 คน?" | "พนักงานเหล่านี้จะถูกย้ายไปยังที่เก็บถาวร จะไม่ปรากฏในรายชื่อที่ใช้งานอยู่อีกต่อไป แต่สามารถกู้คืนได้ในภายหลัง" |
| ตัดการเชื่อมต่อ | "ตัดการเชื่อมต่อระบบเงินเดือน?" | "คุณกำลังจะตัดการเชื่อมต่อระบบเงินเดือน รายการที่รอดำเนินการจะไม่ถูกประมวลผล การเชื่อมต่อใหม่จะต้องยืนยันตัวตนอีกครั้ง" |

### ข้อผิดพลาดการตรวจสอบ (Inline)

สำหรับการตรวจสอบระดับฟิลด์ ให้ข้อความสั้นและตรงประเด็น:

| สถานการณ์ | ข้อความ |
|---|---|
| ฟิลด์ที่จำเป็น | "จำเป็นต้องกรอกฟิลด์นี้" |
| รูปแบบไม่ถูกต้อง | "กรุณากรอกที่อยู่อีเมลที่ถูกต้อง" |
| ข้อผิดพลาดช่วง | "ค่าต้องอยู่ระหว่าง 1 ถึง 100" |
| ยาวเกินไป | "สูงสุด 255 ตัวอักษร" |
| ซ้ำ | "อีเมลนี้ถูกใช้งานแล้ว" |

### กฎข้อผิดพลาด

- อย่าโทษผู้ใช้ ("คุณกรอกผิด" → "กรุณากรอกข้อมูลที่ถูกต้อง...")
- ระบุสิ่งที่ผิดพลาดอย่างเฉพาะเจาะจงเมื่อเป็นไปได้
- เสนอวิธีแก้ไขหรือทางออกเสมอ
- ใช้ตัวพิมพ์แบบประโยค ไม่ใช้ตัวพิมพ์ใหญ่ทั้งหมด
- ไม่แสดงรหัสข้อผิดพลาดในข้อความที่ผู้ใช้เห็น
- การยืนยันการทำลายข้อมูลต้องระบุชื่อรายการที่จะได้รับผลกระทบ
- ปุ่มหลักในกล่องยืนยันต้องตรงกับคำกริยาการดำเนินการ ("ลบ", "เก็บถาวร")

## สถานะว่าง

โครงสร้าง: **ส่วนนี้คืออะไร** + **วิธีเริ่มต้น**

| ส่วน | ข้อความ | การดำเนินการ |
|---|---|---|
| รายชื่อพนักงาน | "ยังไม่มีพนักงาน" | ปุ่ม "Add Employee" |
| คำขอลา | "ไม่มีคำขอลาที่จะแสดง" | คำแนะนำตัวกรอง |
| การแจ้งเตือน | "คุณดูครบทุกรายการแล้ว" | -- |
| ผลการค้นหา | "ไม่พบผลลัพธ์สำหรับ '[query]'" | "ลองปรับการค้นหาหรือตัวกรองของคุณ" |
| แดชบอร์ด | "ไม่มีข้อมูลสำหรับช่วงเวลานี้" | "เลือกช่วงวันที่อื่น" |

## ข้อความตัวยึดตำแหน่ง

- ใช้**ตัวอย่าง** ไม่ใช่คำสั่ง: `"e.g., juan@company.com"` ไม่ใช่ `"Enter your email"`
- ตัวยึดตำแหน่งจะหายไปเมื่อโฟกัส -- อย่าใช้เป็นป้ายกำกับเพียงอย่างเดียว
- ใช้ตัวยึดตำแหน่งสั้นๆ

| ฟิลด์ | ตัวยึดตำแหน่ง |
|---|---|
| ชื่อ | "e.g., Juan dela Cruz" |
| อีเมล | "e.g., juan@company.com" |
| โทรศัพท์ | "e.g., +63 917 123 4567" |
| ค้นหา | "ค้นหาพนักงาน..." |
| วันที่ | "MM/DD/YYYY" |
| จำนวนเงิน | "0.00" |

## การจัดรูปแบบวันที่และเวลา

| รูปแบบ | รูปแบบวันที่ | ตัวอย่าง |
|---|---|---|
| วันที่แบบสั้น | MM/DD/YYYY | 01/15/2025 |
| วันที่แบบกลาง | MMM D, YYYY | Jan 15, 2025 |
| วันที่แบบยาว | MMMM D, YYYY | January 15, 2025 |
| แบบสัมพัทธ์ | X ที่แล้ว | "2 ชั่วโมงที่แล้ว", "เมื่อวาน" |
| เวลา (12 ชม.) | h:mm A | 2:30 PM |
| วันที่ + เวลา | MMM D, YYYY h:mm A | Jan 15, 2025 2:30 PM |
| ช่วงวันที่ | MMM D -- D, YYYY | Jan 15 -- 31, 2025 |

ค่าเริ่มต้น: **MM/DD/YYYY** (มาตรฐานฟิลิปปินส์) ใช้วันที่แบบสัมพัทธ์สำหรับกิจกรรมภายใน 7 วัน

## การจัดรูปแบบตัวเลข

| ประเภท | รูปแบบ | ตัวอย่าง |
|---|---|---|
| จำนวนเต็ม | คั่นหลักพันด้วยจุลภาค | 1,234,567 |
| ทศนิยม | ทศนิยม 2 ตำแหน่ง | 1,234.56 |
| สกุลเงิน (PHP) | ₱ + จุลภาค | ₱1,234,567.00 |
| เปอร์เซ็นต์ | ทศนิยม 1 ตำแหน่ง + % | 85.5% |
| ตัวย่อ | K / M / B | 12.5K, 1.2M |

จัดคอลัมน์ตัวเลขชิดขวาในตาราง ย่อตัวเลขที่มากกว่า 10,000 ในการแสดงผลแบบกะทัดรัด

## เครื่องหมายวรรคตอน

- **มหัพภาค**: ใช้ในประโยคสมบูรณ์ ไม่ใช้ในป้ายกำกับ หัวข้อ ปุ่ม
- **เครื่องหมายอัศเจรีย์**: หลีกเลี่ยง ใช้เฉพาะกรณีเฉลิมฉลองจริงๆ
- **จุดไข่ปลา (...)**: ใช้สำหรับข้อความที่ถูกตัดทอนหรือกำลังโหลดเท่านั้น ไม่ใช้เพื่อทิ้งค้าง
- **เครื่องหมาย &**: หลีกเลี่ยงในข้อความ ใช้ได้ใน UI ที่มีพื้นที่จำกัด
