# การเข้าถึง

Sprout Design System มีเป้าหมายการปฏิบัติตาม **WCAG 2.1 Level AA** ส่วนประกอบทั้งหมดได้รับการทดสอบด้วย `@axe-core/playwright`

## คอนทราสต์สี

### อัตราส่วนขั้นต่ำ

| องค์ประกอบ | อัตราส่วน | นำไปใช้กับ |
|---|---|---|
| ข้อความปกติ | **4.5:1** | ข้อความที่เล็กกว่า 18px (หรือ 14px ตัวหนา) |
| ข้อความขนาดใหญ่ | **3:1** | ข้อความ 18px ขึ้นไป (หรือ 14px ตัวหนาขึ้นไป) |
| ส่วนประกอบ UI | **3:1** | เส้นขอบ ไอคอน ตัวบ่งชี้โฟกัส |
| องค์ประกอบที่ไม่ใช่ข้อความ | **3:1** | องค์ประกอบแผนภูมิ วัตถุกราฟิก |

### คอนทราสต์โทเค็นหลัก

| โทเค็น | บนพื้นหลัง | ผ่านหรือไม่? |
|---|---|---|
| `spr-text-color-strong` (mushroom-950) | white-50 | ผ่าน |
| `spr-text-color-base` (mushroom-600) | white-50 | ผ่าน |
| `spr-text-color-weak` (mushroom-400) | white-50 | ตรวจสอบตามบริบท |
| Focus ring (mushroom-900) | พื้นหลังใดก็ได้ | ผ่าน (3:1 ขึ้นไป) |

::: warning
อย่าพึ่งพาสีเพียงอย่างเดียวในการสื่อข้อมูล ให้จับคู่กับไอคอน ป้ายข้อความ หรือรูปแบบเสมอ
:::

## การนำทางด้วยแป้นพิมพ์

### ลำดับการแท็บ

- ลำดับการแท็บต้องเป็นไปตามลำดับการอ่านตามสายตา (ซ้ายไปขวา บนลงล่าง)
- ใช้ลำดับ DOM ตามธรรมชาติ -- หลีกเลี่ยงค่า `tabindex` ที่มากกว่า 0
- องค์ประกอบแบบโต้ตอบทั้งหมดต้องเข้าถึงได้ด้วยแป้นพิมพ์

### การกำหนดปุ่มลัด

| ปุ่ม | การดำเนินการ |
|---|---|
| `Enter` | เปิดใช้งานปุ่ม ส่งฟอร์ม ยืนยันกล่องโต้ตอบ |
| `Space` | สลับช่องทำเครื่องหมาย สวิตช์; เปิดใช้งานปุ่ม |
| `Escape` | ปิดโมดัล ป็อปโอเวอร์ ดรอปดาวน์ |
| `Arrow keys` | นำทางภายในรายการ เมนู กลุ่มปุ่มตัวเลือก แท็บ |
| `Tab` | ย้ายไปยังองค์ประกอบที่โฟกัสได้ถัดไป |
| `Shift + Tab` | ย้ายไปยังองค์ประกอบที่โฟกัสได้ก่อนหน้า |
| `Home` | ย้ายไปยังรายการแรกในรายการ/เมนู |
| `End` | ย้ายไปยังรายการสุดท้ายในรายการ/เมนู |

### การจัดการโฟกัส

- **วงแหวนโฟกัส**: `box-shadow: 0px 0px 0px 2px #394141` (mushroom-900)
- **โมดัล**: ต้องกักโฟกัส -- วนรอบระหว่างองค์ประกอบที่โฟกัสได้ตัวแรกและตัวสุดท้าย
- **เมื่อปิด**: คืนโฟกัสไปยังองค์ประกอบที่เรียกโมดัล
- **ลิงก์ข้าม**: จัดเตรียม "ข้ามไปยังเนื้อหาหลัก" เป็นองค์ประกอบที่โฟกัสได้ตัวแรก

## รูปแบบ ARIA

### ปุ่ม

```html
<!-- ปุ่มไอคอนอย่างเดียว: ต้องมี aria-label -->
<spr-button variant="ghost" aria-label="Delete employee">
  <Icon icon="ph:trash" />
</spr-button>
```

### โมดัล

```html
<div role="dialog" aria-modal="true" aria-labelledby="modal-title">
  <h2 id="modal-title">Confirm Deletion</h2>
  <!-- โฟกัสถูกกักไว้ภายใน -->
</div>
```

### แท็บ

| องค์ประกอบ | Role | แอตทริบิวต์ |
|---|---|---|
| คอนเทนเนอร์แท็บ | `tablist` | -- |
| แท็บ | `tab` | `aria-selected`, `aria-controls` |
| แผงแท็บ | `tabpanel` | `aria-labelledby` |

### ฟิลด์ฟอร์ม

| แอตทริบิวต์ | เมื่อใดที่ใช้ |
|---|---|
| `aria-required="true"` | ฟิลด์ที่จำเป็น |
| `aria-invalid="true"` | ฟิลด์ที่มีข้อผิดพลาดในการตรวจสอบ |
| `aria-describedby` | เชื่อมโยงไปยังข้อความช่วยเหลือ/ข้อผิดพลาด |
| `aria-errormessage` | เชื่อมโยงไปยังคำอธิบายข้อผิดพลาด |

### พื้นที่แบบสด

| ส่วนประกอบ | Role / แอตทริบิวต์ |
|---|---|
| Snackbar | `role="alert"` (assertive) |
| สถานะการโหลด | `aria-live="polite"` |
| การตรวจสอบฟอร์ม | `aria-live="polite"` บนคอนเทนเนอร์ข้อผิดพลาด |

## แนวทางโปรแกรมอ่านหน้าจอ

### รูปภาพและไอคอน

- **รูปภาพตกแต่ง**: `alt=""`
- **รูปภาพให้ข้อมูล**: ข้อความ `alt` ที่อธิบายรายละเอียด
- **ไอคอนตกแต่ง** (อยู่ข้างข้อความ): `aria-hidden="true"`
- **ไอคอนแบบเดี่ยว** (ปุ่มไอคอนอย่างเดียว): `aria-label` บนปุ่ม

### จุดสังเกต

| จุดสังเกต | องค์ประกอบ | การใช้งาน |
|---|---|---|
| ส่วนหัว | `<header>` | ส่วนหัวเว็บไซต์ ด้านบนเมนูด้านข้าง |
| การนำทาง | `<nav>` | เมนูด้านข้าง เบรดครัมบ์ แท็บ |
| เนื้อหาหลัก | `<main>` | พื้นที่เนื้อหาหลัก |
| ส่วนท้าย | `<footer>` | ส่วนท้ายของหน้า |

ใช้ `aria-label` เพื่อแยกความแตกต่างระหว่างองค์ประกอบ `<nav>` หลายตัวในหน้าเดียวกัน

## เป้าหมายการสัมผัส (มือถือ)

| คุณสมบัติ | ค่า |
|---|---|
| ขนาดขั้นต่ำ | **44 x 44px** (WCAG 2.5.8) |
| แนะนำ | 48 x 48px |
| ระยะห่างระหว่างเป้าหมาย | ขั้นต่ำ 8px |

## การลดการเคลื่อนไหว

```css
@media (prefers-reduced-motion: reduce) {
  *, *::before, *::after {
    animation-duration: 0.01ms !important;
    transition-duration: 0.01ms !important;
  }
}
```

- ปิดใช้งานแอนิเมชันตกแต่ง
- คงไว้ซึ่งการเคลื่อนไหวที่จำเป็น (สปินเนอร์ แถบความคืบหน้า)
- การเฟดแบบความทึบเพียงอย่างเดียวเป็นทางเลือกที่ยอมรับได้

## กฎ

| กฎ | รายละเอียด |
|---|---|
| WCAG 2.1 AA | เป้าหมายการปฏิบัติตามขั้นต่ำ |
| เข้าถึงได้ด้วยแป้นพิมพ์ | องค์ประกอบแบบโต้ตอบทั้งหมดเข้าถึงได้ด้วยแป้นพิมพ์ |
| โฟกัสที่มองเห็นได้ | ตัวบ่งชี้โฟกัสต้องมีคอนทราสต์ 3:1 |
| สีเพียงอย่างเดียวไม่เพียงพอ | ให้จับคู่สีกับไอคอน ข้อความ หรือรูปแบบเสมอ |
| ต้องมีป้ายกำกับ | ฟิลด์ฟอร์มทุกตัวต้องมีป้ายกำกับที่เชื่อมโยง |
| การเชื่อมโยงข้อผิดพลาด | ข้อความข้อผิดพลาดเชื่อมโยงผ่าน `aria-describedby` |
| การกักโฟกัส | โมดัลต้องกักและคืนโฟกัส |
| ทดสอบด้วย axe | ใช้ `@axe-core/playwright` สำหรับการตรวจสอบอัตโนมัติ |
